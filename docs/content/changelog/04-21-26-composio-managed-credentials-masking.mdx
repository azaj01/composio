---
title: "Automatic Credential Masking for Composio-Managed Connections"
description: "Access and refresh tokens for connections on Composio-managed auth configs are now always masked in API responses, regardless of project-level masking. Connections on your own auth config are unchanged."
date: "2026-04-21"
---

For connections created through a Composio-managed auth config, access tokens, refresh tokens, and the `Authorization` header are now always masked in `connected-accounts` API responses — regardless of your project's `mask_secret_keys_in_connected_account` setting.

**Most integrations need no changes.**

- Tool execution through the SDK, Tool Router, and MCP is unchanged. Composio continues to handle authentication server-side, so your agents and workflows never needed to read these tokens directly.
- Connections on your own auth config are unchanged. Token visibility continues to follow your project's masking setting exactly as before.

## What is a Composio-managed auth config?

When you connect a user through a toolkit's default integration — for example, Gmail or Slack without registering your own OAuth application — the connection runs through an OAuth application that Composio owns and operates on behalf of every team using that default. The access and refresh tokens the provider issues are bound to Composio's shared OAuth client.

You can tell a connection is Composio-managed by the `auth_config.is_composio_managed: true` field on the connected-account resource.

## Why this change

Because the OAuth application is shared across every team on that default, any action a provider takes on it — rate limits, scope adjustments, or in the extreme case, suspending the OAuth client — affects everyone using it. Keeping the credentials inside Composio protects the reliability of the shared integrations, and tool execution continues to work without any change; Composio uses the tokens server-side.

## What changes in the API

| Connection type | Project masking setting | Tokens in response |
|---|---|---|
| Composio-managed | On | Masked (unchanged) |
| Composio-managed | Off | **Masked (new behavior)** |
| Your own auth config | On | Masked (unchanged) |
| Your own auth config | Off | Returned in full (unchanged) |

### Example

Before, with project masking disabled on a Composio-managed Gmail connection:

```json
{
  "state": {
    "val": {
      "access_token": "ya29.a0ARrdaM9...full token...",
      "refresh_token": "1//04...full token..."
    }
  }
}
```

After:

```json
{
  "state": {
    "val": {
      "access_token": "ya29...",
      "refresh_token": "1//0..."
    }
  }
}
```

The same treatment applies to the `data`, `params`, and `headers.Authorization` fields wherever these values would otherwise appear.

### Affected endpoints

- [`GET /api/v3/connected_accounts/{id}`](https://docs.composio.dev/rest-api/connected-accounts/get-connected-accounts-by-nanoid)
- [`GET /api/v3/connected_accounts`](https://docs.composio.dev/rest-api/connected-accounts/get-connected-accounts)
- [`POST /api/v3/connected_accounts`](https://docs.composio.dev/rest-api/connected-accounts/post-connected-accounts)

## If you need raw tokens

To keep reading `access_token` or `refresh_token` from API responses, connect through your own auth config. Tokens issued against an OAuth application you own are returned in full (subject to your project's masking setting).

1. Register an OAuth application with the provider — for Gmail, start in the [Google Cloud Console](https://console.cloud.google.com/apis/credentials).
2. [Create a custom auth config](https://docs.composio.dev/rest-api/auth-configs/post-auth-configs) using those credentials.
3. Create new connections against that auth config.

<Callout type="info">
**No action is needed for most teams.** If you build on the Composio SDK, Tool Router, or MCP, your integrations keep working as-is. If your application reads `access_token` or `refresh_token` directly from a Composio-managed connection, moving those connections onto a custom auth config preserves the previous response shape.
</Callout>
